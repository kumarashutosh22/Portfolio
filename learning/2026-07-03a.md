# Learning Materials

During this session, we discussed several architectural and tooling decisions. Here are the seminal resources to review so you understand *why* we are doing things this way:

1. **Static Analysis Reality Check (ESLint vs. Semgrep)**
   - You requested Semgrep for a static portfolio. Semgrep is for complex security vulnerabilities in backend services or intricate data flows. For a static React site, it is textbook over-engineering.
   - Read: [ESLint Getting Started](https://eslint.org/docs/latest/use/getting-started) - This is what you actually need for linting React code.

2. **CSS Foundations**
   - We are using Vanilla CSS instead of Tailwind to keep the bundle size small and demonstrate actual frontend competence.
   - Read: [MDN: The Cascade](https://developer.mozilla.org/en-US/docs/Web/CSS/Cascade) - Understand how CSS actually works before relying on utility classes.

3. **Git for Windows**
   - [Git Download](https://git-scm.com/download/win) - The fundamental tool for version control. Always ensure it is in your system PATH.
